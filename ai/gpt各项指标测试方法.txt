# 一、仪器能力速览（关键参数与结论）

1. RIGOL DHO814 示波器（你有的 DHO814）

   * 4 通道，**100 MHz 带宽**，12-bit，**最高实时采样 1.25 GSa/s（单通道模式）**，最大内存深度 **25 Mpts**，高捕获率模式可达百万波形/s。([tw.rigol.com][1])
   * 结论：非常适合做**侧信道功耗轨迹采集（DPA/TVLA）**：采样率与内存深度可捕获几十 µs — 数十 ms 的运算窗口，并在 1.25 GSa/s 下获得充足时间分辨率（注意：100 MHz 带宽会限制 >100 MHz 的频谱分量，但对 MCU 功耗 SCA 常见频谱通常足够）。高 SNR、12-bit 有利于微小泄露检测。

2. RIGOL DM858 数字万用表（台式 DMM）

   * 5½ 位，**最大读数速率 \~125 readings/s**，大容量日志（\~500k 点）。([Batronix][2])
   * 结论：用于**稳态/平均电流/能耗统计**与长时间平均测量非常合适，但**不适合捕获瞬态功耗轨迹**（采样太慢，无法做 DPA/TVLA 所需的高分辨率追踪）。

3. GW Instek GPD-4303S 可编程直流电源

   * 多路、线性/可编程、分辨率与测量功能可到 mV/mA 级，适合作为 DUT 的稳定电源与供电源监测基准。不同文档给出较好输出稳定性与分辨率（见手册）。([GW Instek][3])
   * 结论：用于给 CM32M433R 板供电与做整体功耗基线（在不需要高时间分辨率的场合）很合适；但其自带电流显示/测量通常受响应时间与分辨率限制，不替代示波器+分流测量用于 SCA。

4. ATTEN 焊台

   * 用于电路改造（安装采样电阻、跳线、探针接地、焊接分流器等）。
   * 结论：必需，用来改装电源回路、安装低阻分流并做 PCB 修改。

5. 重要参考（侧信道/TVLA 指南）

   * TVLA/侧信道检测的统计阈值、采样与样本数等有成熟建议（例如 t-test 门限常用 4.5，需做两组独立测试，通常需要数千〜数万条轨迹用于 DPA/TVLA）。详见 NIST 和相关侧信道工作组资料。([Rambus][4])

# 二、对关键测试的可测性、方法与精度评估（逐项）

> 注：下面给出**实务级测量配置**（你可以直接按这个做）。我同时把**哪些附件是必须/建议/可选**用注释标出（因为你之前说不确定是否有分流电阻/电流探头）。

---

## A. 总能耗 / 平均功耗 / 每操作能耗（mJ/KB）

* **可测性**：可行。
* **推荐仪器与方法**：

  1. 使用 **GPD-4303S** 给开发板供电（稳定电压）。读取电源显示作为粗略基线（好用于快速调整）。([Newark Electronics][5])
  2. 为精确测量能耗，采用 **分流电阻 + 示波器测量电流波形并做积分**：将小电阻 $R_s$（建议 0.1 Ω — 1 Ω，见下）串联在 Vcc 的低侧或高侧（**推荐低侧接地回路**更安全，但注意地回路的测量方法），用 DHO814 在分流两端测电压 $V_s$，瞬时电流 $I(t)=V_s/R_s$，瞬时功率 $P(t)=V(t)\cdot I(t)$（若 V 是板供电电压，可近似 P(t)=Vcc\*I(t)），对每次操作周期积分得到能量。
  3. 用 **DM858** 做长期平均电流/能耗对比、日志记录（用于验证平均值与示波器积分结果）。([Batronix][2])
* **必要 / 建议附件**：

  * 必需：低阻分流电阻（0.1 Ω 精密电阻 若要获得较低压降；或 1 Ω 获得更大电压但注意压降）；*若使用 0.1 Ω，通常需放大器（见下）*。
  * 推荐：差分探头或电压放大器（差分/隔离放大器）用于测量分流电阻两端小电压，或使用示波器的差分测量（若无差分探头，可用两个通道并做数学差分，但注意接地限制）。
* **估计精度**：

  * 使用 0.1 Ω + 示波器（12-bit）并外接差分放大器，能达到 **约 ±3–8%**（取决于放大器噪声、分流电阻精度与触发/对齐）。
  * 仅用 GPD 显示或 DM858 作平均测量：**±1–3%**（在稳态测量下），但不能得到每次操作的瞬时能量分布。
* **常见参数选择举例（计算示意）**：

  * 估算 MCU 峰值电流 100 mA，使用 1 Ω 分流 → 峰电压 100 mV（示波器直接可量测）；使用 0.1 Ω → 峰电压 10 mV（需增益 10–100 放大器）。
  * DHO814 垂直灵敏度可到 500 μV/div，12-bit 分辨率，适配低电压测量但需要低噪差分测量链。([tw.rigol.com][1])

## B. 侧信道轨迹采集（TVLA / DPA / CPA）

* **可测性**：在现有 DHO814 基础上**可行且有效**（但效果取决于是否配备差分探头 / 低噪放大器 /良好分流布线）。
* **推荐仪器与方法**：

  1. **测量点**：在 DUT 的 Vcc 供电回路中串联低阻分流电阻（推荐 0.1–1 Ω），在分流上测得电压波形即代表电流波形。
  2. **探测与输入**：最佳做法是使用 **差分电流探头**（示波器电流探头）或**低噪差分放大器 + 示波器**，避免接地回路注入噪声；如果没有差分探头，可把分流放在地线上并用示波器单端测量（注意：单端测量 ground reference 可能引入较大噪声）。
  3. **触发**：用板上 GPIO（例如在 crypto 函数入口/出口打脚）作为外部触发输入到示波器（或给示波器一个 trigger pulse），以实现每次操作波形对齐。
  4. **采样设置**：在 DHO814 上建议：单通道或两通道模式使用 **1.25 GSa/s**（对高分辨率），内存深度按需设置（例如 25 Mpts 可在 1.25 GSa/s 下捕获 20 ms，足够长）。通道带宽 100 MHz；如需更高频成分（>100 MHz），该示波器有限制，但对 MCU SCA 通常 100 MHz/1.25 GSa/s 已足够。([tw.rigol.com][1])
  5. **样本数**（Trace 数量）：TVLA 初测建议 **NA = NB = 5k \~ 10k**（即总 10k–20k traces）；若要做 DPA/CPA 密钥恢复，通常需要 **10k–50k** traces（依实现与 SNR 而定）。参照 TVLA 文献与实践建议。([Rambus][4])
* **必要 / 建议附件**：

  * 强烈建议：**差分电流探头（或差分探头 + 低噪放大器）**；若没有，至少用高精度分流 +差分放大器（例如 ×10–×100）以提高 SNR。
  * 还需：探针夹、良好接地、屏蔽以及用于触发的逻辑线。ATTEN 焊台用于安装分流电阻和打点。
* **估计精度 / 可检出最小泄露**：

  * 使用 DHO814 + 差分探头 + 1.25 GSa/s，**在良好屏蔽与正确触发条件下**，能检测到典型 MCU 中基于寄存器位的泄露（TVLA t-test 能降到 <4.5）并能复现实验室级别的 DPA。总体 SNR 与最小可检出幅度受采样噪声、放大器增益与分流选择影响；在理想配置下，**可将最小可检出泄露幅度降低到数 μV（经放大后）对应的电流变化**。
  * 若没有差分探头，只用单端分流测量，噪声会高，可能需要 **更多 traces（数倍）** 才达到同等统计显著性。

## C. 时序 / 响应延时（加密/签名执行时间）

* **可测性**：易测。示波器加上触发（GPIO）或使用板上计时（GPIO 切换）均可高精度测量。DM858 无法测此类短时时延。
* **推荐方法**：用 MCU 在开始/结束位置拉低/拉高一个测量 GPIO（或在函数入口/出口 toggling），示波器测量两次边沿间时间。DHO814 的时间基与采样足够精确，精度可以到 **sub-µs** 级甚至 **ns** 级（取决于采样率与片上触发点的抖动）。
* **估计精度**：在 1.25 GSa/s 下，**时间分辨率约 0.8 ns**（1 / 1.25e9），实际测量误差主要来自触发抖动与 GPIO 播种，通常能达到 **≤ ±0.1 µs** 精度（对几十 ms 的签名耗时评估而言精度充足）。([tw.rigol.com][1])

## D. 随机数熵 / TRNG 测试（NIST 风险/统计）

* **可测性**：需软件工具（样本采集 + 统计测试的软件），仪器只负责输出数据源。DM858 与示波器不适合熵测试。
* **推荐方法**：读取 TRNG 输出的大量样本（≥几 MB），用 NIST SP 800-22、Dieharder、ENT 等工具做统计测试。GPD 电源 /示波器 无关此测试。
* **估计精度**：统计测试本身给出 p-value 与通过/不通过结论；硬件设备不会限制结论，只要样本量足够（数百万位）。

# 三、必需/推荐的额外附件清单（按重要性）

1. **差分电流探头（强烈推荐）** 或等效电流探头（示波器厂商或第三方）。——用于高 SNR 的瞬态电流测量（DPA/TVLA）。
2. **差分放大器 / 电流放大器（×10–×100）**（如 INA、低噪仪用放大器），当分流电阻较小或没有差分探头时使用。
3. **精密分流电阻**（0.1 Ω、1 Ω，1%/0.1% 精度各一只），以及焊接座/插座以便快速切换。
4. **短吸收式屏蔽盒 / 金属屏蔽**（减少外部 EMI，提高 SNR）。
5. **触发线 / 逻辑电平探针**（用于把 MCU 的 start/stop GPIO 拉到示波器触发输入）。
6. （可选）高带宽示波器差分探头或功率解析仪，如 Keysight/TEK 的功率探头，用于更精细的频谱分析（如果后续需要更高带宽或更好的 SNR）。

> 如果你目前**没有差分探头或放大器**，测试仍然可行——只是要用更大的分流电阻（如 1 Ω）并把其放于地线上使用单端测量，并且 **需要更多 trace 数量与更严格的屏蔽** 来弥补噪声；我在下面的表格会把“是否需要附件”标注清楚。

# 五、实际操作要点（步骤样板，直接可用）

1. **硬件改造（用 ATTEN 焊台）**

   * 在开发板 Vcc 的 Ground 返回线上焊接精密分流电阻（建议先用 1 Ω 测试、确认无功能影响，再改为 0.1 Ω 提高动态线性）。
   * 为保证测量安全与可重复性，使用短导线、良好焊接、并在附近预留触发 GPIO 线位。

2. **示波器连接（DHO814）**

   * 若有差分探头：直接夹在分流电阻两端（或 Vcc 回路上），设置探头带宽 ≥100 MHz（或按探头规格）。
   * 若无差分探头：用两个通道分别测分流两端并在示波器上做数学差分（注意接地回路限制与安全）。
   * 设置采样：1.25 GSa/s（或在多通道下 625 MSa/s），内存深度按操作时长设（例如 25 Mpts 可存 20 ms @1.25 GSa/s）。触发用外部 GPIO 脉冲。([tw.rigol.com][1])

3. **TVLA / DPA 数据采集**

   * 采集 NA = NB = 5000 traces 初测（若噪声大逐步增加到 10k–50k）。
   * 每两组数据独立采集两次 TVLA（符合 Goodwill/TVLA 建议）。([NIST Computer Security Resource Center][6])

4. **能量计算与精度控制**

   * 在积分前对分流波形做 DC 偏移校正与基线滤波，利用已知 Vcc 值计算能量。重复多次取平均并用 DM858 做长期平均值核验。

### （三）指标说明（基于 CM32M433R 的调整依据）

1. **加密性能**

   * 由于主频提升至 144 MHz，相对于 100 MHz 平台可提高约 1.4× 的性能，因此硬件 AES 可目标定在 ≥ 100 Mbps；
   * Ascon 软件实现可利用 RISC-V 指令优化（如位操作、交叉寄存器）目标定在 ≥ 8 Mbps；
   * 掩码防护引入的性能损耗仍应控制在较低范围（≤ 10 %）。

2. **签名性能**

   * 在更高主频下，SLH-DSA 的签名 + 验证时间能相对降低，因此签名生成目标设为 ≤ 40 ms，验证为 ≤ 25 ms；
   * 私钥派生等辅助操作目标设为 ≤ 30 ms。

3. **能耗预算**

   * 依据资料，该芯片动态功耗约 125 µA/MHz，即在 144 MHz 下为 \~18 mA（活动状态下）([inf.news][2])；取合理利用率后，加密 + 签名模式下平均功耗设定为 ≤ 35 mW 是一个较为保守、可达成的目标；
   * 每 KB 加密所消耗的能耗目标设为 ≤ 0.20 mJ/KB，是在较低功率平台下的节能设定。

4. **安全 / 侧信道防护**

   * TVLA / DPA 指标原则上不依赖平台，只要测量环境合理，目标仍保持不变；
   * 随机数质量、盲化刷新等设计在 RISC-V 平台下实现方式可能与 Cortex 不同，但指标可保持一致。
